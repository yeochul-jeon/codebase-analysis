## 세션 7 요약

이 레포 자신(`codebase-analysis`)을 target으로 E2E dogfooding 전 경로 검증 완료.

**발견된 회귀 (2건) 및 수정:**

**BUG-1: `src/` 내 stale `.js` 아티팩트** — Session 5/6 과정에서 이전 빌드 단계가 `packages/server/src/` 하위에 컴파일된 `.js`를 남긴 채 대응 `.ts`가 나란히 생성됨. walker가 두 파일을 모두 인덱싱해 `createApp` 등이 중복 심볼로 나타났다. `.ts` 대응 파일이 있는 11개 `.js` 삭제 + `.gitignore`에 `.codebase-analysis/` 추가.

**BUG-2: FTS5 `MATCH 'pack*'` 경로 오염** — `searchSymbols`가 `symbols_fts MATCH ?`에 `name, kind, signature, file_path` 전 컬럼을 적용하므로 쿼리가 `file_path`의 `packages/...` 경로에 매칭되어 잘못된 대량 결과를 반환. `name:${query}*` 컬럼 스코프로 수정. 추가로 `q` 파라미터를 `z.string().regex(/^[A-Za-z0-9_]+$/)` 로 좁혀 FTS5 쿼리 인젝션 방지. smoke 테스트 51 → 54 checks 추가(path-prefix no-hit, injection→400).

**검증 결과:**
- `analyze .` → 463 symbols / 35 files (stale .js 제거 후), `source.zip` 정상
- `push` → `index_id=1`, 재push → 409 skip (ADR-010 멱등성)
- Web UI: `GET /`, `/style.css`, `/app.js`, `/s/:key` 전부 200, search-form·symbol.html 포함
- REST: search 5종 (`createApp`·`SqliteAdapter`·`registerAnalyzeCommand`·`pack`·`extractFromJS`) 전부 정확, body slice, references, file-overview 정상
- MCP 4 tool: `search_symbols`·`get_symbol_body`·`get_references`·`get_file_overview` 전부 정상 응답, 404 에러 경로 `-32603`으로 매핑 확인
- 에러 경로: 401(wrong token), 400(missing q / injection), 404(unknown repo/key) 전부 정상
- `pnpm typecheck`·`pnpm build` 전체 통과, smoke 54/54

**변경된 파일:**
- `packages/server/src/storage/db/sqlite.ts` — `name:${query}*` 컬럼 스코프
- `packages/server/src/routes/reads.ts` — `q` 파라미터 regex 유효성 검사
